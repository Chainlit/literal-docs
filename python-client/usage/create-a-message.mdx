---
title: "Create a message"
---

Messages are the user-facing part of a [Thread](/concepts/thread). They represent a message from the user, the system or the assistant.

Note that Messages must be created inside a [Thread](/python-client/usage/create-a-thread).

There is only one method to create a message.

We recommend setting at least the `content` (message text), `type` (user type) and `name` (user name) parameters.

```python
client.message(content="Hello World", type="assistant_message", name="Chatbot")
```

By default, the message id is a random uuid. You can set a custom id to match your own system. Message ids must be unique across the messages and steps of your project.

## Message parameters

<ParamField path="thread_id" type="uuid" required>
  The id of the thread
</ParamField>

<ParamField path="id" type="uuid">
  The id of the message. If not provided, a random uuid will be generated.
</ParamField>

<ParamField path="name" type="string">
  The name of the message
</ParamField>

<ParamField path="type" type="MessageStepType" default="undefined">
  The type of the message. A Message can be one of the following types:

- `system_message`: A system message (e.g. a static welcome message)
- `user_message`: A message sent by the user
- `assistant_message`: A message sent by the assistant

</ParamField>

<ParamField path="metadata" type="dict" default="{}">
  Metadata associated with the message. This enables you to add custom fields to
  your messages.
</ParamField>

<ParamField path="parent_id" type="uuid">
  The id of the parent message. This enables you to keep track of the parent
  step.
</ParamField>

<ParamField path="timestamp" type="string">
  The timestamp when the message was created.
</ParamField>

<ParamField path="created_at" type="string">
  The server-side creation time of the message.
</ParamField>

<ParamField path="content" type="string">
  The content of the message.
</ParamField>

<ParamField path="tags" type="List[str]" default="[]">
  The tags of the message. This is a complimentary field to the metadata field.
  It enables you to add custom tags to your messages.
</ParamField>

<ParamField path="feedback" type="Feedback">
  The feedback object associated with the message.
</ParamField>

<ParamField path="attachments" type="List[Attachment]" default="[]">
  The attachments associated with the message.
</ParamField>
